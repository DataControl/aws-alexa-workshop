6. Create the Alexa backend

It is now time to create the backend for your skill, ie the service that will receive the Alexa directives corresponding to the user interactions with Alexa. In order to create a robust solution without having to setup and maintain a lot of infrastructure you are going to use Lambda.

Make sure you’ve selected the N.Virginia region before you continue.

Create Lambda Execution Role

  1. Go to IAM Console, choose Roles on the left side bar

  2. click Create Role

  3. Under Choose the service that will use this role, choose Lambda

  4. Click Next: Permissions, Next: Tags, Next: Review

  5. In the Review page, enter alexa-lambda-role for the Role name, and choose Create role

  6. Click the alexa-lambda-role, under Permissions tab, click Add inline policy

  7. In Create Policy page, select JSON, and copy & paste the following policy. Please remember to replace <device-table-name>. Check your DynamoDB table name in DynamoDB Console.

    {
"Version": "2012-10-17",
"Statement": [
    {
        "Action": [
            "dynamodb:Query"
        ],
        "Resource": [
            "arn:aws:dynamodb:*:*:table/<device-table-name>/index/ByUsernameThingName"
        ],
        "Effect": "Allow"
    },
    {
        "Action": [
            "iot:UpdateThingShadow"
        ],
        "Resource": [
            "*"
        ],
        "Effect": "Allow"
    }
]
}

  8. Click Review Policy

  9. In the Name field, input a name for this policy

  10. Click Create Policy

  11. Click Attach Policy

  12. Search for LambdaBasic. Select the checkbox on AWSLambdaBasicExecutionRole

  13. Click Attach policy

Create the Lambda function

  1. Go to Lambda Console, click Create function

  2. Select Author from scratch, and enter the following information:

    • Name: Provide a name for your Lambda function
    • Runtime: To use the sample code provided in this topic, choose Node.js 10.x
    • Role: under Perission, select Use an existing role and select the role you previously created

  3. Click Create Function. Your function should be created and you will move to Configuration

  4. In the Configuration tab, expand the Designer section, and click on Add triggers

  5. Select the Alexa Smart Home trigger

  6. In the Configure triggers section, add the Skill ID from the developer console in the box specified.

  7. Leave Enable trigger checked. This enables the Amazon Alexa service to call your Lambda function.

  8. Click Add

Upload Code

  1. Got to the Cloud9 instance and clone this repo,

    git clone https://github.com/massi-ang/alexa-iot-smarthome alexa-iot-smarthome
cd alexa-iot-smarthome
cd src

  2. Edit src/config.json file. You find the necessary information in AWS Console and Alexa Console

  3. Run npm install --production to install dependencies

  4. While in the /src folder, make a zip file to include index.js, auth.js, config.json, alexa/ and node_modules/, these files/directories should be located at the root level of the zip file

    zip -r ../lambda.zip index.js auth.js config.json alexa/ node_modules/

  5. Download the lambda.zip file to your laptop (right-click on the file name in the explorer pane)

  6. Go to AWS Lambda Console, click the lambda function

  7. Under Function code, click Upload to upload the zip file

  8. Click the Save button at the top right corner

Configure Alexa Skill Endpoint

  1. From the Lambda console, copy the ARN of the Lambda Function in the top right corner

  2. Go to Alexa Developer Console, select the created Smart Home Skill

  3. Paste the Lambda ARN in Default endpoint and in the North America endpoint

  4. Click the Save button in top right corner
